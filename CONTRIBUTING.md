# Contributing & Collaboration

## Session workflow
1. Pull latest changes: `git pull`.
2. Work only in the session's target file(s) per `meta/session_plan.md`.
3. Use clear, conventional commits (feat/fix/docs/chore).
4. Update `meta/version_log.md` with a one‑line change summary.
5. Push changes; tag versions when a session is complete.

## Safety & Privacy
- This repo is private; verify collaborator list regularly.
- Avoid PHI where possible; de‑identify or use year/month granularity.
- Store PDFs/scans under `data/lab_results/` and avoid committing raw sensitive data unless necessary.
